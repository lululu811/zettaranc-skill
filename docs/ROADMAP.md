# 产品路线图

> 最后更新：2026-07-11  
> 当前版本：v3.9.0

## 版本号规定

- **MAJOR**：不兼容 API 变更（心智模型重构、CLI 不兼容变更）
- **MINOR**：向后兼容的功能新增（新战法/指标/CLI 子命令/数据源）
- **PATCH**：bug 修复、技术债清理、内部重构

> 严格遵循语义化版本（Semantic Versioning）。技术债清理属于 PATCH，不是 MINOR。

## 阶段一：多策略融合（v3.10.x）

**目标**：将模拟器层的战法共振能力接入组合回测和验收管线

### v3.10.0 多策略融合引擎

**目标**：`PortfolioBacktestEngine` 从 B1-only 改造为多策略并行

**改动点**：
- `modules/verify/portfolio_engine.py` 的 `_scan_and_buy()` 方法
- 复用 `simulator/resonance_scorer.py` 的共振评分逻辑
- 支持 B1 + B2 + 长安 + 沙漏等策略，按市场环境动态调整权重

**验收标准**：
- [ ] 支持至少 3 种策略并行选股
- [ ] 策略权重可配置（按市场环境）
- [ ] 回测结果展示各策略贡献度
- [ ] 测试覆盖率 > 80%

**预计工期**：2-3 周

**改动点**：

**验收标准**：
- [x] ATR 动态止损（`method="atr_based"`，止损距离 = ATR × multiplier）
- [x] 移动止损（trailing stop，`trailing_stop_enabled` 开关）

**预计工期**：1 周 → 已完成

## 技术债务

### 高优先级

1. **文档完善** ⏳
   - `docs/USER_GUIDE.md` 需更新反映 v3.8+ 新功能
   - 缺少 API 文档（各模块公开接口）

2. **性能优化** ⏳
   - `precompute_market_contexts()` 大数据量时较慢
   - `get_kline_dicts()` 批量查询缺优化

### 中优先级

3. **类型注解** ⏳
   - 部分模块缺少类型注解
   - mypy 未全量开启

4. **错误处理** ⏳
   - API 调用失败时静默返回 None
   - 缺少统一错误码

### 低优先级

5. **依赖升级** ⏳
   - Python 3.10 → 3.12
   - pandas 版本升级

6. **CI/CD 优化** ⏳
   - 缺少自动化部署
